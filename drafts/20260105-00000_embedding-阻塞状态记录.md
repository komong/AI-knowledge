# AI Knowledge Embedding 模型加载阻塞 — 状态记录

## 一、问题描述

**症状**：`import sentence_ transformers` 在 Windows .venv 中被系统静默杀死（exit code 1，无报错），后续所有 embedding 流程无法跑通。

**影响**：草稿无法 embedding 入库，MCP Server 无法启动。

### 关键发现
- `torch` + `transformers` 能单独 import，`sentence_ transformers` import 时被静默终止
- 即使关闭杀软，PyTorch 的 C++ 扩展(.pyd) 文件可能在杀软隔离区，需要**杀软重启**才能解除
- `echo nigel | sudo -S` 在 PowerShell 中无法自动填入 WSL sudo 密码（交互式提示被触发）
- `astral.sh` 域名在 WSL DNS 中无法解析，无法用 curl 安装 uv

## 五、推荐解法路径

### 方案 A：Windows 改用 ONNX 路径（推荐，最快）

用 `optimum` 将 BGE-M3 导出为 ONNX格式，ONNX Runtime 不依赖 PyTorch C++ 扩展，完全规避杀软问题：

```bash
pip install optimum[onnxruntime]
python -m optimum.onnxruntime.export --model BAAI/bge-m3 --task feature-extraction
```

修改 `embedder. py` 改用 ONNX Runtime 推理。

### 方案 B：杀软白名单（需用户操作）

让用户在杀软中手动添加排除项：
- `d:\Desktop\Sadbox\AI Knowledge\.venv\Lib\site-packages\torch\lib\torch_cpu. pyd`
- `d:\Desktop\Sadbox\AI Knowledge\.venv\Lib\site-packages\sentence_transformers`
- `d:\Desktop\Sadbox\AI Knowledge\.venv\Scripts\python.exe`

### 方案 C：WSL 独立环境

在 WSL 里手动跑一次：
```bash
wsl bash -c "sudo apt install python3.12-venv"  # 输入密码 nigel
wsl python3 -m venv ~/ai-venv
wsl ~/ai-venv/bin/pip install torch FlagEmbedding qdrant-client
```

然后 MCP Server 跑在 WSL 里，通过 localhost 与 Windows Qdrant 通信。
