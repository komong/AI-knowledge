# 问题

在 WSL 里建 Python venv + 装 torch + 下 BGE-M3 模型的链路整条跑下来经常卡死（apt 无响应 / pip 下载几十分钟 / snapshot_download 限速），需要可规避的方案。

# 场景

Windows 11 + WSL2 Ubuntu（nigel@DESKTOP-VQ82ABC），Qdrant 已在 WSL Docker 正常跑在 `localhost:6333`。Windows `.venv` 里 BGE-M3 模型（~4.58GB）和 torch wheel 已下好，但 `sentence_transformers` 被杀软拦截 `.pyd`，导致 Windows 侧 embedding 链路不通。原计划迁 WSL，但 WSL 全新装环境容易卡死。

# 卡点定位

| 症状 | 根因 |
|------|------|
| `apt install` 无响应 | Ubuntu 默认源慢 / DNS 抖动 |
| `pip install torch` 几十分钟 | 默认拉 CUDA 版 >2GB |
| `snapshot_download` BGE-M3 卡 | HF 直连 / hf-mirror 限速 |
| venv 激活后 import 卡 | 首次加载模型属正常，非卡死 |

# 解决方案

## 方案 4（首选，0 新增下载）：Windows 杀软白名单 + 重启

根因是 PyTorch `.pyd` 被杀软隔离，关杀软不释放，必须重启杀软才解除。由于 `embedder.py` 本就用 `FlagEmbedding`（非 `sentence_transformers`），大概率直接通。

**操作步骤**：
1. 杀软里加白名单（3 条）：
   - `d:\Desktop\Sadbox\AI Knowledge\.venv\Lib\site-packages\torch\`
   - `d:\Desktop\Sadbox\AI Knowledge\.venv\Lib\site-packages\sentence_transformers\`
   - `d:\Desktop\Sadbox\AI Knowledge\.venv\Scripts\python.exe`
2. **重启杀软**（不是关闭，隔离区需重启才释放）
3. 在 Windows 验证：
   ```powershell
   py -c "from FlagEmbedding import BGEM3FlagModel; import torch; print('ok', torch.__version__)"
   ```
4. 通过后直接 `py -m scripts.commit --dry-run`

**验收**：`embed_text('hello')` 输出 1024 维向量。

## 方案 1（备用）：WSL 复用 Windows 已下的缓存

如果方案 4 不通再走 WSL，但复用 Windows 已下文件避免重下：

```bash
# 挂 Windows 的 HF 缓存到 WSL
export HF_HOME=/mnt/c/Users/<用户名>/.cache/huggingface
# pip 共享 Windows wheel 缓存
pip install torch --index-url https://download.pytorch.org/whl/cpu \
  --cache-dir /mnt/d/pip-cache
```

## 方案 2（备用）：WSL 换国内源 + 仅装 CPU torch

```bash
# apt 换清华源
echo nigel | sudo -S sed -i 's|archive.ubuntu.com|mirrors.tuna.tsinghua.edu.cn|g' /etc/apt/sources.list
echo nigel | sudo -S apt update
echo nigel | sudo -S apt install -y python3.12-venv python3-pip

# pip 换源
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

## 方案 3（备用）：WSL 最小依赖，不跑 `pip install -e .`

`-e .` 会把 mcp、qdrant-client、FlagEmbedding、pydantic、python-frontmatter 全拉一遍容易卡。先只装 embedding 必需 4 件：

```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install FlagEmbedding qdrant-client python-frontmatter
# 验 embedder 能跑，再装 mcp
```

# 已踩过的坑（避免重踩）

- `echo nigel | sudo -S` 在 PowerShell 调 `wsl` 时密码灌不进去 → 必须先 `wsl` 进交互 shell 再执行
- WSL 默认 Python 不含 pip，`python3 -m pip` 报 No module → 先 `apt install python3-pip`
- `astral.sh` DNS 在 WSL 不通 → 不用 uv，走 apt + pip
- `use_fp16=True` 在 CPU 会崩 → `embedder.py` 需动态判 `torch.cuda.is_available()`
- hf-mirror 对 `.DS_Store` 返 403 → `snapshot_download(..., ignore_patterns=['*.DS_Store','imgs/*'])`

# 推荐路径

**方案 4 → 方案 1+2+3 组合**。先试 Windows 杀软白名单（5 分钟见分晓、0 下载），失败再迁 WSL。
