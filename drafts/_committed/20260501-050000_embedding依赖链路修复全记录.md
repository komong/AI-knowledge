# 问题

`from FlagEmbedding import BGEM3FlagModel` 在 Windows `.venv` 中无任何报错直接静默退出（exit=-1073741819 / 0xC0000005），导致 `src/embedder.py` 无法加载，commit 入库与 MCP Server 全部阻塞。

# 场景

- Windows 11 + WSL2（Qdrant 跑在 WSL Docker `localhost:6333`，Python `.venv` 跑在 Windows）
- CPU：Intel i7-8550U（支持 AVX2）
- Python：`.venv\Scripts\python.exe`（uv 创建）
- 杀软：火绒
- 网络：HuggingFace 必须走 `hf-mirror.com`

# 排查链路（7 层剥洋葱）

按时间顺序逐层定位，每层一个独立根因：

| # | 症状 | 根因 | 解法 |
|---|---|---|---|
| 1 | `import torch` 静默 kill | 火绒 HIPS 拦截 `.pyd` | 火绒"信任区"加项目目录，**重启火绒** |
| 2 | `No module named pip` | uv 建的 venv 默认不装 pip | `python.exe -m ensurepip --upgrade` |
| 3 | `sentence_transformers` ImportError | `transformers 5.7.0` 要求 `huggingface-hub<1.0`，当前 1.13.0 | 降 transformers 到 `<5.0`（4.57.6） |
| 4 | `huggingface-hub` 版本冲突 | `transformers 4.x` 要求 `hub<1.0`，pip 自动解析又装回新版 | 显式 `pip install "huggingface-hub>=0.34.0,<1.0" --force-reinstall --no-deps` |
| 5 | `pyarrow` access violation 0xC0000005 | **numpy 2.4.4** 与老 pyarrow 编译时的 ABI 不兼容（`numpy.core.multiarray failed to import`） | 降 `numpy<2` + `pyarrow==15.0.2`（`--force-reinstall --no-deps`） |
| 6 | `AttributeError: pyarrow 'json_'` | `datasets 4.x` 调用 `pa.json_()`（pyarrow 18+ API），与 pyarrow 15 不匹配 | 降 `datasets==2.21.0` |
| 7 | 加载 BGE-M3 时 `403 Forbidden imgs/.DS_Store` | hf-mirror 对 `.DS_Store` 返 403（已知老坑） | `snapshot_download('BAAI/bge-m3', ignore_patterns=['*.DS_Store', 'imgs/*'])` 预下载 |

# 解决方案

## 1. 火绒放行（HIPS 是静默 kill 的元凶）

- 火绒主界面 → 病毒查杀 → 设置 → 信任区 → 添加文件夹：`d:\Desktop\Sadbox\AI Knowledge\`
- 如果菜单叫"访问控制 / 程序执行控制"，为 `.venv\Scripts\python.exe` 加"全部允许"自定义规则
- **退出火绒进程或重启**（仅关闭防护不够，隔离区需重启释放）

## 2. venv 补 pip

```powershell
.\.venv\Scripts\python.exe -m ensurepip --upgrade
```

## 3. 对齐整套依赖版本（关键组合，**别再乱升**）

```powershell
cd "d:\Desktop\Sadbox\AI Knowledge"
.\.venv\Scripts\python.exe -m pip install "transformers<5.0" --force-reinstall --no-deps
.\.venv\Scripts\python.exe -m pip install "huggingface-hub>=0.34.0,<1.0" --force-reinstall --no-deps
.\.venv\Scripts\python.exe -m pip install sentence-transformers --force-reinstall --no-deps
.\.venv\Scripts\python.exe -m pip install "numpy<2" "pyarrow==15.0.2" --force-reinstall --no-deps
.\.venv\Scripts\python.exe -m pip install "datasets==2.21.0" --no-deps
```

## 4. 预下载 BGE-M3 模型（跳 403）

```powershell
$env:HF_ENDPOINT="https://hf-mirror.com"
.\.venv\Scripts\python.exe -c "from huggingface_hub import snapshot_download; snapshot_download('BAAI/bge-m3', ignore_patterns=['*.DS_Store', 'imgs/*'])"
```

## 5. 验证链路

```powershell
.\.venv\Scripts\python.exe -c "from FlagEmbedding import BGEM3FlagModel; print('fe ok')"
.\.venv\Scripts\python.exe -c "from src.embedder import embed_text; v=embed_text('hello'); print('dim=', len(v))"
```

预期 `dim= 1024`。

# 关键诊断技巧

- `exit=-1073741819` = `0xC0000005` = **C 层 Access Violation**（不是 Python 异常，是 DLL 崩溃）
- 用 **`python -X faulthandler -c "..."`** 打印崩溃时的 C 栈，瞬间定位挂在哪个 `.pyd`
- 静默退出（无报错直接回 prompt）一般是 HIPS 拦截 or 二进制 ABI 不兼容，两者区分方法：
  - 火绒退出后仍崩 → ABI
  - 火绒退出后变正常 → HIPS

# 已踩过的坑（避免重踩）

- 不要用全局 `py`，必须 `.\.venv\Scripts\python.exe`
- `--force-reinstall` 必须搭配 `--no-deps`，否则 pip resolver 会把依赖又拖回新版本导致冲突
- 降 datasets 后若再跑 `pip install -e .` 会把 datasets 升回 4.x，破坏环境
- `ignore_patterns` 写法：`['*.DS_Store', 'imgs/*']`（两条都要）
